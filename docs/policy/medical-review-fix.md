검수 반려는 콘텐츠를 처음부터 다시 만들라는 뜻이 아니에요. 어느 위치가 왜 문제인지 확인하고, 같은 의미의 위험 표현이 다른 위치에 남아 있지 않게 고치는 과정이에요.

## 먼저 반려 위치를 나눠요

| 반려 위치 | 확인할 것 |
| --- | --- |
| 제목 | 과장 표현, 가격만 강조한 표현, 시술명이 불명확한 표현 |
| 상세 설명 | 가격 조건, 부작용, 개인차, 적용 제외 조건 누락 |
| 이미지 | 이미지 안 가격, 과장 문구, 선정성, 결과 보장처럼 보이는 구성 |
| 옵션 | 옵션명, 옵션가, 포함 항목, 제외 항목 불일치 |
| 상담 조건 | 앱 표시 조건과 실제 상담 조건이 다른 경우 |

## 반려 사유별로 이렇게 수정해요

| 반려 사유 | 수정 방법 |
| --- | --- |
| 효과 보장으로 보여요 | 결과를 단정하지 말고 개인 상태에 따라 달라질 수 있다고 바꿔요 |
| 가격 조건이 부족해요 | 대표가, VAT, 옵션, 추가 비용 가능성, 적용 기간을 나눠 적어요 |
| 이미지 문구가 달라요 | 이미지 파일 자체를 교체하거나 입력값과 같은 문구로 수정해요 |
| 부작용 안내가 부족해요 | 통증, 붓기, 출혈, 염증, 개인차 가능성을 상세 설명에 추가해요 |
| 의료진 경력이 과장돼 보여요 | 확인 가능한 경력만 남기고 주관적 표현은 삭제해요 |
| 선정적으로 보여요 | 과도한 신체 노출, 불쾌감을 줄 수 있는 구도, 성적 강조 이미지를 제거해요 |
| 환자 경험담처럼 보여요 | 특정 환자의 결과를 일반화하지 않게 문장을 바꿔요 |

## 수정할 때 자주 하는 실수

- 제목만 고치고 이미지 안 문구를 그대로 둬요.
- 대표 가격만 바꾸고 옵션 가격과 VAT를 그대로 둬요.
- 효과 보장 문장을 다른 표현으로 돌려 말하지만 의미는 그대로 남겨요.
- 반려된 이벤트를 새로 만들어 같은 문구를 다시 넣어요.
- 상담팀에 수정 내용을 공유하지 않아 상담에서 이전 조건을 안내해요.

## 재검수 요청 전 순서

1. 반려 사유를 제목, 본문, 이미지, 가격, 옵션으로 나눠요.
2. 지적된 위치만 고치지 말고 같은 표현이 다른 위치에 남아 있는지 확인해요.
3. 이미지 파일 안 문구까지 확인해요.
4. 상담팀에 변경된 가격과 조건을 공유해요.
5. 광고에 연결된 이벤트라면 광고 노출 상태도 같이 확인해요.

## 반려가 반복될 때

반려가 반복되면 문장 하나의 문제가 아니라 이벤트 구조가 불명확한 경우가 많아요. 대표가, 옵션가, 적용 조건, 추가 비용, 부작용 안내를 다시 분리해서 환자가 오해하지 않게 정리해 주세요.
