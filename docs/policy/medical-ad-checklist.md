이 문서는 이벤트, 병원 소개, 의사 소개, 이미지, 배너를 등록하기 직전에 확인하는 체크리스트예요. 등록 담당자가 이 순서대로만 확인해도 반려와 상담 혼선을 줄일 수 있어요.

## 1. 제목을 확인해요

제목은 환자가 목록에서 가장 먼저 보는 정보예요. 시술명, 상담명, 적용 범위가 드러나야 하고 결과를 보장하는 표현은 피해야 해요.

| 확인 항목 | 등록 기준 |
| --- | --- |
| 시술명 | 환자가 이해할 수 있는 실제 진료·수술 명칭을 사용해요 |
| 과장 표현 | 최고, 유일, 완벽, 보장처럼 결과를 단정하는 표현은 쓰지 않아요 |
| 가격 강조 | 가격만 앞세우지 말고 적용 조건이 본문에 연결돼 있어야 해요 |
| 대상 표현 | 파트너센터에 표시되는 대상 조건만 사용해요 |
| 기간 표현 | 이벤트 기간이 있으면 시작일과 종료일을 정확히 맞춰요 |

## 2. 가격과 옵션을 확인해요

가격은 대표가, 옵션가, 추가 비용 가능성을 나눠서 봐요. 상담에서 다른 가격이 안내되면 환자 불만으로 이어질 수 있어요.

| 항목 | 꼭 확인할 내용 |
| --- | --- |
| 대표가 | 환자가 처음 보는 기준 가격이에요. 실제 적용 가능한 조건인지 확인해요 |
| VAT | 포함 또는 별도 여부를 명확히 표시해요 |
| 옵션 | 재료, 범위, 방식, 사후관리 등 가격 차이가 나는 이유를 적어요 |
| 추가 비용 | 개인 상태에 따라 추가될 수 있는 항목은 상담 필요 문구와 함께 안내해요 |
| 이미지 가격 | 이미지 안의 가격과 입력칸 가격이 같은지 확인해요 |

## 3. 효과와 회복 표현을 확인해요

남성수술은 개인 상태, 수술 범위, 회복 과정에 따라 결과가 달라질 수 있어요. 효과를 보장하는 문장보다 상담이 필요하다는 문장으로 바꿔요.

| 쓰면 위험한 방향 | 안전한 작성 방향 |
| --- | --- |
| 결과를 확정하는 표현 | 개인 상태에 따라 결과가 달라질 수 있다고 안내해요 |
| 통증이나 부작용을 없다고 단정 | 통증, 붓기, 출혈, 염증 등 가능성과 상담 필요성을 안내해요 |
| 회복 기간을 단정 | 회복 기간은 개인차가 있을 수 있다고 적어요 |
| 만족도를 보장 | 기대 효과와 제한 사항을 구분해요 |

## 4. 이미지와 배너를 확인해요

이미지는 문구보다 더 빠르게 환자에게 인식돼요. 선정성, 가격 불일치, 결과 보장처럼 보이는 이미지는 반려나 수정 요청으로 이어질 수 있어요.

| 확인 항목 | 등록 기준 |
| --- | --- |
| 신체 노출 | 성기사진, 과도한 노출, 불쾌감을 줄 수 있는 이미지는 사용하지 않아요 |
| 전후 비교 | 특정 결과를 보장하는 방식으로 보일 수 있어 별도 검토가 필요해요 |
| 가격 문구 | 입력 가격, VAT, 옵션 조건과 이미지 문구가 같아야 해요 |
| 의료진 사진 | 실제 의료진 자료만 사용하고 경력 표현을 과장하지 않아요 |
| 병원 이미지 | 실제 병원 시설과 다르게 보정하거나 다른 장소 이미지를 쓰지 않아요 |

## 5. 상담팀 공유 전에 확인해요

콘텐츠는 검수만 통과하면 끝나는 게 아니에요. 상담팀이 앱에 등록된 조건과 같은 기준으로 안내해야 해요.

1. 이벤트 제목과 상담 스크립트의 시술명이 같은지 확인해요.
2. 대표가, 옵션가, VAT, 추가 비용 가능성을 상담팀이 알고 있는지 확인해요.
3. 예약 가능 시간과 실제 진료 가능 시간이 맞는지 확인해요.
4. 부작용과 개인차 안내를 상담에서 생략하지 않게 공유해요.
5. 반려 후 수정한 내용은 상담팀에 다시 알려요.

## 등록 전 최종 확인

| 질문 | 예 / 아니오 |
| --- | --- |
| 앱에 보이는 가격과 상담에서 안내할 가격이 같나요? | 아니오라면 등록 전 수정해요 |
| 이미지 안 문구와 입력 문구가 같나요? | 아니오라면 이미지 또는 입력값을 수정해요 |
| 부작용과 개인차 안내가 있나요? | 아니오라면 상세 설명에 추가해요 |
| 환자가 결과를 보장받는다고 오해할 표현이 없나요? | 아니오라면 문구를 바꿔요 |
| 상담팀이 같은 조건으로 응대할 수 있나요? | 아니오라면 등록 전 내부 공유가 먼저예요 |
